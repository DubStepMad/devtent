# DevTent License (DTCL v1.0)

DevTent uses the **DevTent Community License (DTCL) v1.0** — a copyleft license that keeps the software free **and** bans anyone from selling it.

## You CAN

- Use DevTent for any purpose, including commercial software development
- Modify the source code for your needs
- Share copies with others **at no charge**
- Fork the project and publish your fork as open source under DTCL v1.0
- Contribute improvements via pull requests

## You CANNOT

- **Sell DevTent** — no paid downloads, subscriptions, license keys, or paywalls. Ever.
- **Sell a fork or derivative** — any project built from DevTent code cannot be sold either
- **Charge for hosted access** — no "DevTent Pro" SaaS, paid cloud dev environments, or similar
- **Keep derivatives closed source** — if you share a modified version, source must be public under DTCL v1.0
- **Bundle it in a paid product** — if access to DevTent (or a derivative) is part of what customers pay for, that's a sale

## What about using DevTent to build paid apps?

That's fine. DTCL prohibits selling **the DevTent software itself**, not applications you build *with* DevTent. You can run a Laravel e-commerce site developed locally in DevTent and charge customers for your product — you just can't charge for DevTent.

## What about donations?

Unconditional donations to support development are OK, as long as donors don't get exclusive access, features, or copies that others can't get for free.

## Why not plain AGPL?

AGPL allows selling copies (with source provided). DTCL adds an absolute **no-sale** rule on top of copyleft so the core dev environment stays free for everyone.
