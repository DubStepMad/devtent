Getting started

  1. On first run, enable Install recommended stack in setup — or use Quick Add manually
  2. Start services from the toolbar or Services
  3. Create or drop projects in Projects / www/
  4. Sync virtual hosts and open myapp.test in your browser

Turn services on to add them to your Procfile, then start or stop them. Install missing runtimes in Quick Add first.

    Install runtimes via Quick Add, then enable services with the switches above.

    Service output, per-site web server logs, and DevTent app diagnostics. Files are stored in logs/ (app log in AppData).

    Select a log file to view output.

    If *.test sites do not resolve, click Update hosts file (Admin). Windows should show a UAC prompt — check the taskbar if you do not see it. DevTent itself stays non-admin.
